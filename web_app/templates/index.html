about

The lighting_project is an IoT contraption that is designed to show off the core components of IoT. It features a smart light stick that can be connected to the Internet and is controllable remotely, allowing users to change its behaviour to show different light patterns, as well as make use of different inputs to generate interesting patterns.

how it works

node

The node consists of a DotStar LED Strip which is connected to a Hibiscus Sense ESP32 microcontroller. The Hibiscus is capable of WiFi connectivity and also possesses a built-in MPU6050 accelerometer and gyroscope. Using these capabilities, the Hibiscus is able to connect to the LAN and communicate with the edge device wirelessly, as well as easily provide data on the forces applied to the light stick.

The Hibiscus is programmed to convert color data received from the edge device into output on the Dotstar LED strip, and deliver accelerometer data to the edge device for processing.

edge

The edge device is responsible for the communication between the node and the cloud, where it receives commands from the remote user, downloads new image/audio files, and uploads sensors data into the cloud. This is achieved by using MQTT, a lightweight publish/subscribe messaging protocol, to communicate the required payloads with minimal network bandwidth.

This is also where most of the processing occurs, as the edge device has to process incoming input such as accelerometer data, image/audio file data and microphone audio, and convert them into color data that is understandable by the Hibiscus, which is then sent to the Hibiscus through WiFI.

cloud

The cloud service used in this project is Amazon Web Services (AWS). AWS provides a myriad of services, but only IoT Core, DynamoDB, S3 and EC2 are used for this project.

IoT Core is used as the intermediary between the node and the cloud storage in the form of DynamoDB and S3. On one end, after receiving payload data from the node, making use of the provided Rules Engine, IoT Core will automatically route the data to DynamoDB and save them. On the other end, after receiving commands from the user, IoT Core will update the node's state in the cloud, before sending the data down to the node for it to process.

DynamoDB is used to save sensors data from the node, as mentioned previously. This data is then pulled into this website to be displayed, as can be seen in Lightsaber Mode.

S3 is used to store user uploaded image and audio files. When new files are uploaded, S3 will automatically notify IoT Core, which will then be passed down to the corresponding node to inform them of the new file.

EC2 is used to launch server instances on the cloud, which in this context, is used to host the web server for this web interface.

All of these different parts work together to form the entire cloud infrastructure, which allows for the remote operation of the light sticks.
